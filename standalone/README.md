# Roam Editable Table

一个面向 Roam Research 的单文件可编辑表格脚本。

它不会改造 Roam 原生 `{{table}}`，而是提供一个独立的 editable table 组件。用户在块中输入 `{{editable-table}}` 或 `{{[[editable-table]]}}` 后，脚本会自动在该块下初始化并渲染一张可直接编辑、可增删行列、并持久化保存到底层 Roam blocks 的表格。

## 当前能力

当前版本支持：

- 识别宿主块：`{{editable-table}}` / `{{[[editable-table]]}}`
- 自动初始化默认 3x3 表格
- 同页多实例支持
- 单元格点击后直接进入编辑
- 表头编辑
- 单元格多行编辑与保存
- `blur` 保存
- `Ctrl+Enter` / `Cmd+Enter` 保存当前编辑
- `Tab / Shift+Tab` 保存并切换到相邻单元格
- `Escape` 取消本次编辑
- 增加行
- 删除当前选中行
- 在当前选中列后增加列
- 删除当前选中列
- 页面刷新、切页后自动重新渲染
- 单行 JSON 损坏时局部报错，不影响整页

## 数据结构

宿主块内容：

- `{{editable-table}}`
- 或 `{{[[editable-table]]}}`

脚本会在宿主块下维护以下结构：

- `table-meta:: {"version":1,"cols":[{"id":"c1","label":"A"},{"id":"c2","label":"B"},{"id":"c3","label":"C"}]}`
- `table-rows::`
  - `"[\"\",\"\",\"\"]"` 形式的行数据块

说明：

- `table-meta::` 保存列定义
- `table-rows::` 是行容器
- 每一行都是 `table-rows::` 下的一个子 block
- 每一行 block 的内容是一个 JSON array
- 表头文字保存在 `table-meta:: cols[].label`
- 单元格内容保存在对应行 JSON array 的同索引位置

这样做的好处是：

- 表头和数据行的持久化边界清晰
- 修改单元格时只需要更新对应行 block
- 增删列时只需要更新 meta 和所有行数组
- 旧数据不需要迁移

## 实现思路

### 1. 独立渲染，不改造原生 `{{table}}`

插件只识别特殊 marker：

- `{{editable-table}}`
- `{{[[editable-table]]}}`

识别到 marker 后，在该块下插入自定义表格 UI，而不是接管 Roam 原生表格能力。

### 2. 扫描 + 观察器双保险

`extension.js` 同时使用：

- `MutationObserver`
- 定时 polling

这样在以下场景里都能自动挂载：

- 页面首次加载
- 页面切换
- block 懒加载
- DOM 重新渲染

### 3. UI 只是数据的投影

渲染时会从 Roam blocks 读取当前表格状态，再重新生成表格 DOM。也就是说：

- Roam block 数据是事实来源
- UI 只是当前状态的展示层
- 编辑、增删行列后会重新读取并渲染

### 4. 行列操作的持久化方式

- **增加行**：在 `table-rows::` 下创建新的子 block
- **删除行**：删除当前选中行对应的 row block
- **增加列**：更新 `table-meta:: cols`，并给每一行数组插入空字符串
- **删除列**：更新 `table-meta:: cols`，并从每一行数组删除对应索引

当前实现中：

- 新增行会插入到当前选中行之后；如果当前没选中 body 行，则追加到底部
- 新增列会插入到当前选中列之后；如果当前没有有效列选中，则追加到最右侧
- 删除行只对当前选中的 body 行生效
- 删除列对当前选中的 header/body 所在列生效

## 安装

当前交付是单文件脚本：

- `standalone/extension.js`

也就是说，这个 README 对应的是现在放在 `standalone/` 子目录下的独立版本。

注意：**仅仅把文件放到某个目录里并不会生效**，必须确保这段 JS 在 Roam 页面里被真正执行。

你需要把这个脚本加载到自己的 Roam 环境中。常见方式包括：

- 通过你现有的自定义 JS 注入方案加载
- 通过浏览器用户脚本方式加载
- 后续再包一层 Roam Depot 插件结构

如果你是手动测试，至少要确认两件事：

1. 打开 Roam 页面后，这个文件已经实际执行。
2. 浏览器控制台里执行 `window.__roamEditableTablePlugin` 能看到插件对象，而不是 `undefined`。

如果这里是 `undefined`，说明不是插件逻辑没工作，而是脚本根本没有被加载执行。

## 使用方式

1. 确保 `extension.js` 已被加载到 Roam。
2. 在任意页面插入一个块，输入：
   - `{{editable-table}}`
   - 或 `{{[[editable-table]]}}`
3. 脚本会自动在其下创建默认 3x3 表格。
4. 点击任意单元格或表头即可开始编辑。
5. 使用上方工具栏进行增删行列。

## 交互说明

### 单元格编辑

- 单击单元格：选中并进入编辑
- 单击表头：选中并进入表头编辑
- `Ctrl+Enter` / `Cmd+Enter`：保存当前编辑
- `Tab`：保存并移动到下一个单元格
- `Shift+Tab`：保存并移动到上一个单元格
- `Escape`：取消当前编辑并恢复旧值
- `blur`：离开输入框时自动保存

### 文本行为

- 单元格支持多行文本
- 换行会保存在对应 row JSON 中
- 点击已有内容的单元格时，不会全选文本，光标会落在文本末尾

### 行列工具栏

- `Add row`：在当前选中行后插入一行；无行选中时追加到底部
- `Delete row`：删除当前选中的 body 行
- `Add column`：在当前选中列后插入一列
- `Delete column`：删除当前选中的列

禁用规则：

- 只剩 1 行时，`Delete row` 禁用
- 只剩 1 列时，`Delete column` 禁用
- 当前选中表头时，`Delete row` 禁用
- 如果存在损坏的 row JSON，列增删会禁用
- 加载中或结构修改中，所有结构操作按钮会禁用

## 开发过程中遇到的主要问题

下面是这次实现里实际遇到、并已处理的关键问题。

### 1. marker 已输入，但页面没有表格

现象：

- 输入 `{{editable-table}}` 后没有反应
- `window.__roamEditableTablePlugin` 已经存在

原因：

- 仅仅脚本存在不代表已经正确挂载到对应 block
- 初期实现里，DOM 上拿到的 block 标识和真实 Roam block uid 混淆了

处理：

- 改为用更可靠的 UID 映射和宿主块识别逻辑
- 扫描可见 marker block，再回到 Roam API 读取真实 block 数据

### 2. 能渲染表格，但点击单元格只高亮，不进入编辑

原因：

- 初期点击逻辑只做了 selection，没有立即进入编辑态

处理：

- 点击命中 cell 后，除了 `selectCell()`，还会直接触发 `beginEdit()`

### 3. 单元格只能单行，不能保存换行

原因：

- 早期是单行输入控件，回车行为也被当成提交

处理：

- 改成 `textarea`
- 保留多行文本
- 改为 `Ctrl+Enter` / `Cmd+Enter` 保存
- `Tab / Shift+Tab` 负责保存并切换单元格

### 4. 获取焦点后的高亮边框被内容区域遮住

现象：

- 有文字的区域边框高亮不完整
- 空白区域能看到高亮，文字区域看起来像被白色背景盖住

处理：

- 不再依赖普通边框表现选中态
- 改成通过单元格伪元素 `::after` 绘制覆盖层边框
- 这样高亮不会被内部编辑框背景压住

### 5. 表格最开始只能固定 3x3

原因：

- 初始 schema 创建时把 3 行 3 列写死

处理：

- 保留 3x3 作为首次初始化默认值
- 之后通过工具栏支持动态增删行列
- 底层存储结构仍然保持 `meta + rows`，不需要迁移旧数据

### 6. 增加行/列最开始只能追加到最后

需求后来调整为：

- 新行要插入到当前选中行之后
- 新列要插入到当前选中列之后
- 删除也应作用在当前选中行/列

处理：

- 新增行时根据当前选中行计算插入位置
- 新增列时根据当前选中列计算插入位置
- 删除逻辑始终基于当前选中 spec

### 7. 点击工具栏按钮要点两次才生效

原因：

- 第一次点击先触发了当前编辑单元格的 `blur`
- `blur` 导致保存和重渲染
- 重渲染把第一次点击吃掉了

处理：

- 在 `mousedown` 阶段先拦住工具栏按钮的焦点切换
- 让按钮点击不会先被旧单元格的 `blur + render` 吃掉

### 8. 从一个单元格切换到另一个单元格时，也要点两次

原因：

- 与工具栏问题类似，第一次点击先被旧单元格失焦的重渲染消耗

处理：

- 对 cell 的 `mousedown` 也做了提前拦截
- 让切换焦点时第一次点击就能命中目标单元格

### 9. 点击已有内容的单元格时，文本会被整段选中

原因：

- 进入编辑态时用了 `input.select()`

处理：

- 改成把光标移动到文本末尾
- 更符合继续输入的使用习惯

### 10. 选中表头后，`Delete row` 没有立刻禁用

原因：

- 按钮状态之前主要依赖重新渲染时更新
- 选中状态变化和工具栏禁用态之间存在一拍延迟

处理：

- 增加 `updateToolbarState()`
- 在 `selectCell()` 内立刻同步工具栏状态

## 范围限制

本版本仍不包含以下功能：

- 增强原生 `{{table}}`
- 原生表格导入/转换
- 公式
- 富文本单元格
- Markdown 解析
- CSV / Excel 粘贴导入
- 样式主题配置
- 拖拽调整列宽
- 合并单元格
